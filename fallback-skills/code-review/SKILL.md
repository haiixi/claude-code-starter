# Code Review

## 审查维度

1. **正确性**：逻辑是否正确，边界情况是否处理
2. **安全性**：是否有注入、越权、硬编码密钥等
3. **可维护性**：命名、结构、重复、复杂度
4. **测试**：是否有测试，测试是否合理
5. **性能**：是否有明显性能问题

## 安全扫描重点

- 硬编码密钡 / token / 密码
- SQL 注入（字符串拼接）
- 命令注入（`os.system`、`subprocess shell=True`）
- 不安全反序列化（`pickle.loads`）
- `eval()` / `exec()` 调用
- 路径穿越

## 输出格式

| 优先级 | 问题 | 位置 | 建议 |
|--------|------|------|------|
| P0 | ... | ... | ... |
| P1 | ... | ... | ... |
| P2 | ... | ... | ... |

P0 = 必须修复，P1 = 建议修复，P2 = 可以记录
