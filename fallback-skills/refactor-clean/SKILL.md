# Refactor Clean

## 清理范围

1. 删除未使用的 import / require
2. 删除未引用的函数和变量
3. 删除临时文件和日志文件
4. 合并重复代码
5. 删除已注释掉的代码块
6. 移除调试用的 console.log / print

## 注意事项

- 清理前确认当前分支干净或已提交
- 每次只做一个类别的清理
- 清理后运行测试
- 不要删除可能被其他代码动态使用的内容

## 工作流程

1. 先运行静态分析工具（如 eslint）查找明显问题
2. 手动检查并移除无用代码
3. 运行测试确保没有破坏功能
4. 生成简要的变更摘要
