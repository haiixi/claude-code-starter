# Coding Standards

## 何时使用

- 完成一段代码后想确认是否符合规范
- 审查他人代码时
- 重构前评估代码质量

## 检查清单

1. 命名是否清晰？
2. 函数是否单一职责？
3. 是否有重复代码？
4. 错误处理是否完善？
5. 是否有硬编码魔法值？
6. 注释是否说明"为什么"？
7. 是否有未使用的导入或变量？
8. 是否有明显的性能问题？

## 输出格式

以清单形式列出发现的问题，按严重程度排序：

- 🔴 严重：影响正确性或安全
- 🟡 警告：影响可维护性
- 🟢 建议：可优化的小问题

对于每个问题，给出具体位置和修复建议。
